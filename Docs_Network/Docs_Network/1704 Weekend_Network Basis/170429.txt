	Subnetting시 Network ID 증가
	Summary시 Network ID 감소


	172.17.0.0/24	172.17.0.1/24	172.17.0.2/24	172.17.0.3/24

	172.17.00000000.0 /24
	172.17.00000001.0 /24
	172.17.00000010.0 /24
	172.17.00000011.0 /24

	동일한 주소(172.17.000000까지)는 Mask로 사용
	255.255. 111111 00.0 /22 (<-/24)
	실제 주소의 변화는 없으나, 라우팅 과정에서, 22번째 Bit까지만 확인하도록 유도하는 것.
	
	=> 172.17.0.0
	    255.255.252.0 (/22)

	A: 가능하다.
	00000101         5.1 (Ping)
	00000100         4.0 (Static)
	11111100         252.0 (Subnet)
	----- Network ID가 Match되는 한, Router 통신 가능

	11111100 (252.0)	11111111 (255.0)

	공통된 Bit는 4번째까지이므로, 앞에서 4번째까지 Network ID로 축약이 가능하나,
	불필요한 Network 정보까지 포함되는 것을 방지하기 위하여 (00010000 or 00011111 등)
	더욱 세분화하여 축약한다.

=>	172.16. 00010100 .1 /24
	172.16. 00010101 .1 /24
	172.16. 00010110 .1 /24
	172.16. 00010111 .1 /24
	172.16. 11111100

	172.16. 00011000 .1 /24
	172.16. 00011001 .1 /24
	172.16. 11111110
	반드시 이렇게 할 필요는 없지만, 네트워크를 더욱 세부적으로 관리하기 위하여 필요.
