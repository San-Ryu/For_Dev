
반이중 통신 CSMA/CD
Carrier : 반송파(전기신호)
Sense : 감지
Mutiple : 다중
Access : 접속
Collision : 충돌
Detect : 탐지

반이중 통신(half Duplex)
    송신(전송), 수신의 역할을 모두 수행
    송/수신을 동시에 처리하지 못한다.
    (무전기)

전이중 통신(full Duplex)
    송신(전송), 수신의 역할을 모두 수행
    송/수신을 동시에 처리할 수 있다.

Ethernet II Header

Preamble : Ethernet Header 정보는 아니다.
           서두 값, 데이터의 전송을 알리기 위해 사용
           신호 동기화를 위하여 사용 된다.
           10101010 * 7Byte, 10101011
Dest Addr : 목적지 물리 장치 주소(MAC Address)
Src Addr : 출발지 물지 장치 주소(MAC Address)
Type : 상위 계층 Protocol 종류
       IP:0x0800, ARP:0x0806
Data : 상위 계층으로 전달 하기 위한 Data
FCS : Frame Check Sequence, 프레임에 대한 오류 검출


Switch 기본 기능
  1. Ethernet Frame의 목적지 MAC Address 학습
  2. 학습한 주소를 바탕으로 해당 Interface로만 전달(포워딩)
  3. 전달 해야하는 Interface를 제외한 나머지는 차단(필터링)
  4. 학습된 주소가 없는 경우 모든 Interface로 전달(플러딩)
  5. 학습된 정보는 일정 시간 동안만 유지(에이징)  


IP Address
  Network ID : 네트워크 영역을 구분하기 위한 식별자
  Host ID : 네트워크 영역안의 특정 Host(Device)를 구분하기 위한
            식별자




Subnetting
  Network 수 기준 분리
    1. 기준 네트워크 주소 확인
    2. 분리 할 네트워크 수 확인
    3. 10진수 체계 주소를 2진수 체계 주소로 변환
    4. 네트워크 수에 맞추어서 subnetmask 변경
    5. 변경된 subnetmask에 맞추어 네트워크 분리(구분)
    6. 2진수 주소를 10진수 주소로 변경

    192.168.100.0
    255.255.255.0
    4개의 네트워크로 분리

    11000000.10101000.01100100.00000000
    11000000.10101000.01100100.01000000
    11000000.10101000.01100100.10000000
    11000000.10101000.01100100.11000000
    11111111.11111111.11111111.11000000

    Net ID bit 1개 추가 -> Network 2개 분리 -> 2^1
    Net ID bit 2개 추가 -> Network 4개 분리 -> 2^2
    Net ID bit 3개 추가 -> Network 8개 분리 -> 2^3
    Net ID bit n개 추가 -> Network 2^n개 분리

네트워크 주소 : Host ID에 해당하는 주소 부분이 전부 0bit로 사용
브로드캐스트 주소 : Host ID에 해당하는 주소 부분이 전부 1bit로 사용
서브넷마스크 주소 : 0bit는 Host ID 식별자
                    1bit는 Network ID 식별자
                    1bit부터 연속적으로 나열, 중간에 0bit가 쓰이면 나머지
                    부분은 반드시 0으로 연속적인 배치가 이루어진다.

  Host 수 기준 분리
    1. 기준 네트워크 주소 확인
    2. 하나의 네트워크 영역에서 사용할 Host 수 확인
    3. 10진수 체계 주소를 2진수 체계 주소로 변환
    4. Host 수에 맞추어서 subnetmask 변경
    5. 변경된 subnetmask에 맞추어 네트워크 분리(구분)
    6. 2진수 주소를 10진수 주소로 변경

    192.168.100.0
    255.255.255.0
    30 Host가 필요

    192.168.100.000 00000
    255.255.255.111 00000

    Host ID bit 1개 확정 -> 가용 Host 수 0개 -> 2^1 - 2
    Host ID bit 2개 확정 -> 가용 Host 수 2개 -> 2^2 - 2
    Host ID bit 3개 확정 -> 가용 Host 수 6개 -> 2^3 - 2
    Host ID bit 4개 확정 -> 가용 Host 수 14개 -> 2^4 - 2
    Host ID bit 5개 확정 -> 가용 Host 수 30개 -> 2^3 - 2
    Host ID bit 6개 확정 -> 가용 Host 수 62개 -> 2^6 - 2
    Host ID bit n개 확정 -> Network 2^n개 분리
